一、最常见的两种图建模方式
1️⃣ 元件图（Component Graph）⭐最常见

节点 = 元件
边 = 电气连接（Net）

例如：

R1 ---- C1 ---- U1

图结构：

Node: R1
Node: C1
Node: U1

Edge:
R1 — C1
C1 — U1

这种建模：

用于 电路拓扑分析

EDA机器学习

故障检测

电路分类

很多论文都是这样做的。

2️⃣ 引脚图（Pin-level Graph）

更精细的建模。

节点 = 引脚（pin）
边 = wire / net

例如：

R1 pin1 ---- net1 ---- C1 pin1
R1 pin2 ---- GND

图：

R1.1 —— net1 —— C1.1
R1.2 —— GND

这种方式用于：

PCB布线

信号完整性

SPICE仿真

二、还有一种更严格的EDA表示

EDA软件内部通常使用：

Bipartite Graph（二分图）

节点分两类：

元件节点 (component)
网络节点 (net)

结构：

component —— net —— component

例子：

R1 —— Net1 —— C1

图：

R1 → Net1
C1 → Net1
R1 → Net2
GND → Net2

完全有必要，而且你这个任务用“纯 LLM 直接生成 dict”很容易在结构一致性、可控性、可验证性上翻车。更稳的路线是：LLM负责理解需求 → 专门的图生成模型负责产出结构（节点/边/坐标）→ 规则/仿真器负责校验。下面给你一套可以直接落地的架构思路（从简单到强）。

1) 先把任务拆成 3 个子任务（这一步很关键）

你现在的输出其实同时包含了三种信息：

拓扑：哪些元件、怎么连（edge1…edgeN）

语义约束：比如“RC低通”“LED指示”“去耦”“整流”等功能对应的常见结构

几何布局：每个元件的 (x, y)

建议拆成流水线（也能端到端）：

阶段A：功能 → 电路草图（netlist/graph）

阶段B：草图 → 边列表（edge序列化）

阶段C：草图 → 坐标布局（placement）

这样你能分别评估：拓扑对不对、布局好不好。

2) 推荐的深度学习架构（强烈建议：异构图 + 约束解码）
架构总览

输入：自然语言 prompt
输出：一个异构图（components/pins/nets）+ 坐标

模块1：文本编码器（LLM/Encoder）

用 LLM 或者轻量 encoder 把 prompt 编码成向量 h_text

重点是抽取：功能类型、必需元件、约束（比如必须接地、必须限流）

模块2：图生成器（Graph Generator）

这里建议用 “约束的图解码器”，而不是随便吐 JSON。

两个靠谱选择：

A. 序列化生成（Graph-as-Sequence）

把电路图表示成 token 序列（类似 SMILES 但用于电路）

用 Transformer 解码

每一步生成：

add_node(type=电阻/电容/LED/电源...)

add_edge(u, v, net_id)

优点：实现简单，训练像语言模型

缺点：容易不满足电气规则，需要强约束

B. 显式图生成（GraphRNN / GAE / Diffusion Graph）

先预测节点集合，再预测邻接矩阵/边集合

对 EDA 很适合做 异构图：

node types: component / pin / net

edge types: has_pin / connected_to

优点：结构更稳定、好加规则

缺点：工程复杂一点

我更推荐 异构图生成：component—net—component 的二分图（或 component/pin/net 三分）。

模块3：布局生成器（Placement / Coordinate Head）

布局本质上像“画图”，你可以单独做一个模型：

输入：图结构 + h_text

输出：每个元件的 (x,y)

可选模型：

GNN + 回归头（简单有效）

E(2)-equivariant GNN（更稳：旋转平移不敏感）

扩散模型（Diffusion）生成布局（高级但效果好）

import json
import random
from typing import Dict, Tuple, List

random.seed(42)

Coord = Tuple[int, int]
Edge = Dict[str, List[int]]  # {"R1":[x,y], "VCC":[x,y]}
Output = Dict[str, Dict[str, List[int]]]  # {"edge1": {...}, ...}

# ----------------------------
# 坐标生成：让布局看起来“像原理图”
# ----------------------------
def snap(v: int, step: int = 10) -> int:
    return (v // step) * step

def rand_coord(xmin=10, xmax=190, ymin=10, ymax=190, step=10) -> Coord:
    x = snap(random.randint(xmin, xmax), step)
    y = snap(random.randint(ymin, ymax), step)
    return x, y

def place_chain(names: List[str], start: Coord = None, dx=40, dy=0) -> Dict[str, Coord]:
    """把一串元件按链式排布，视觉上更像原理图"""
    if start is None:
        start = rand_coord(10, 60, 20, 160)
    x, y = start
    pos = {}
    for i, n in enumerate(names):
        pos[n] = (x + i * dx, y + i * dy)
    return pos

def to_edge(a: str, b: str, pos: Dict[str, Coord]) -> Dict[str, List[int]]:
    ax, ay = pos[a]
    bx, by = pos[b]
    return {a: [ax, ay], b: [bx, by]}

    tpl_led_indicator,
    tpl_rc_lowpass,
    tpl_lc_filter,
    tpl_decoupling,
    tpl_voltage_divider,
    tpl_diode_protection,
    tpl_pullup_button,
]

# ----------------------------
# 生成一条样本
# ----------------------------
def gen_one() -> Dict:
    tpl = random.choice(TEMPLATES)
    prompt, edges, comps = tpl()

    # 布局：对“主链”做链式排布，其它点做轻微偏移
    # 主链：按 edges 的出现顺序抽取一条最长链近似
    chain = []
    for a, b in edges:
        if not chain:
            chain = [a, b]
        else:
            if chain[-1] == a and b not in chain:
                chain.append(b)
            elif chain[-1] == b and a not in chain:
                chain.append(a)

    pos = place_chain(chain, dx=random.choice([30, 40, 50]), dy=0)

    # 其余节点：围绕OUT/MID等节点偏移摆放，或随机摆放
    for c in comps:
        if c not in pos:
            anchor = None
            for cand in ["OUT", "MID", "VCC"]:
                if cand in pos:
                    anchor = pos[cand]
                    break
            if anchor:
                ax, ay = anchor
                pos[c] = (snap(ax + random.choice([-40, -30, 30, 40])), snap(ay + random.choice([-40, -20, 20, 40])))
            else:
                pos[c] = rand_coord()

    # 输出 edge1..edgeN
    out: Output = {}
    for i, (a, b) in enumerate(edges, start=1):
        out[f"edge{i}"] = to_edge(a, b, pos)

    return {"prompt": prompt, "output": out}

# 批量生成
# ----------------------------
def gen_dataset(n: int, path: str):
    with open(path, "w", encoding="utf-8") as f:
        for _ in range(n):
            item = gen_one()
            f.write(json.dumps(item, ensure_ascii=False) + "\n")

加上坐标的主要思路是什么？随机生成吗？不太可能吧？

对，不是纯随机生成。
如果纯随机，问题会很多：

同一个电路每次长得都不像原理图

线会乱飞

元件会重叠

不符合常见阅读习惯（输入在左，输出在右，GND在下）

所以加坐标的主要思路应该是“规则布局”，不是瞎随机。

最核心的思路

你这一步其实是在做：

已知图结构（components + connections）
给每个节点分配一个合理的 (x, y)

这本质上是一个布局问题。

一、最简单可行的方法：基于拓扑的规则布局

以你这个例子：

{
  "components": ["VCC", "R1", "OUT", "C1", "GND"],
  "connections": [
    ["VCC", "R1"],
    ["R1", "OUT"],
    ["OUT", "C1"],
    ["C1", "GND"]
  ]
}

它对应的结构是：

VCC -> R1 -> OUT
              |
              C1
              |
             GND

所以坐标不应该随机，而应该按这个结构摆：

主信号路径横向摆放

支路纵向摆放

GND放在下面

VCC通常在左边或上边

比如就得到：

VCC = (10, 20)

R1 = (40, 20)

OUT = (70, 20)

C1 = (70, 50)

GND = (70, 80)

这就是结构驱动布局。

二、一般怎么做
第一步：先找“主链路”

比如：

[
  ["VCC", "R1"],
  ["R1", "OUT"],
  ["OUT", "C1"],
  ["C1", "GND"]
]

这里最主要的路径其实是：

VCC -> R1 -> OUT

这可以看成主链。

然后：

OUT -> C1 -> GND

是从 OUT 引出的支路。

所以规则就是：

主链：横着排

支路：竖着排

第二步：给主链分配横坐标

例如固定：

起点 x = 10

每个元件间隔 dx = 30

主链的 y = 20

那么：

VCC = (10, 20)

R1 = (40, 20)

OUT = (70, 20)

第三步：给支路分配纵坐标

从 OUT 往下挂：

C1 = (70, 50)

GND = (70, 80)

这就自然了。

三、所以“随机”只应该出现在小范围扰动里

不是完全随机，而是：

规则主导 + 少量随机扰动

例如：

主链仍然横排

但起点可以随机：

(10,20) 或 (20,30)

间距可以随机：

dx = 30 / 40 / 50

支路长度也可以稍微变

这样生成的数据既：

看起来合理

又不至于所有样本一模一样

也就是：

大结构由规则决定，小变化由随机决定

四、你这个任务最适合的布局策略
策略 1：模板式布局

比如 RC 低通固定模板：

VCC -- R1 -- OUT
             |
             C1
             |
            GND

对应固定坐标模式：

VCC = (x, y)
R1  = (x+30, y)
OUT = (x+60, y)
C1  = (x+60, y+30)
GND = (x+60, y+60)

这里 x, y 可以随机选一个起点。

这最简单，也最适合你现在批量造数据。

策略 2：图遍历布局

对一般图做：

从电源节点或输入节点开始 BFS / DFS

同一层放同一列

下一层往右放

从某个点引出的支路往下放

这适合更复杂一点的电路。

例如：

第 0 层：VCC

第 1 层：R1

第 2 层：OUT

第 3 层：LOAD / C1 / 其它分支

然后按层生成坐标：

x = base_x + layer * dx
y = base_y + index_in_layer * dy
策略 3：约束布局

更专业一点会加规则：

GND 必须在最下方

VCC 尽量在最左/最上

同一路径尽量水平

去耦电容紧挨负载或电源节点

避免边交叉

避免节点重叠

这个比较像真正 EDA 工具在干的事。

什么叫“高质量模板”

一个模板至少要满足 4 个条件。

1. 功能单一明确

例如：

“LED 指示电路”

“RC 低通滤波”

“电阻分压输出”

不要一开始就做那种：

“带保护、滤波、稳压、驱动、反馈的复合系统”

太复杂，容易失控。

2. 拓扑标准

模板里的连接关系必须是行业里最常见、最标准的那种。

例如 RC 低通就优先用：

{
  "components": ["VIN", "R1", "OUT", "C1", "GND"],
  "connections": [
    ["VIN", "R1"],
    ["R1", "OUT"],
    ["OUT", "C1"],
    ["C1", "GND"]
  ]
}

而不是一上来整各种变体。

3. 元件角色清楚

每个元件最好有明确角色：

VIN：输入

VCC：电源

GND：地

R1：串联电阻/分压电阻

C1：滤波/去耦电容

OUT：输出

LOAD：负载

这样后面布局和生成 prompt 都容易。

方案三：把模板做成“语法规则”

就是不把模板看成一条死数据，而看成一套可生成规则。

比如“LED 指示”模板可以写成：

VCC -> R -> LED -> GND

VIN -> R -> OUT

MID -> OUT

C → C1 / C2

我建议你现在做“模板库分层”

不要把所有模板混在一起。分 3 层。

第 1 层：原子模板

最小、最基础、最标准。

例如：

电源串电阻

电容到地

电阻到地

二极管串联

开关到地

这些很简单，但非常重要。

第 2 层：功能模板

由原子模板组合成有明确功能的电路。

例如：

LED 指示

RC 低通

上拉按键

去耦

分压

第 3 层：复合模板

后面再做。

例如：

整流 + 滤波

分压 + 比较器输入

稳压 + 去耦 + 负载

你现在只做前两层就够了。

为什么这比直接找“高质量模板大全”靠谱

因为“高质量模板”不是网上抄一堆电路图就行。

对你来说，模板必须满足：

适合文本描述

适合转成 graph

适合转成 edge1、edge2

适合自动布局

适合批量扩增

很多真实电路图虽然正确，但并不适合拿来做你的训练模板。

所以你需要的是：

面向数据生成的模板库
而不是
面向工程设计的完整原理图库

layout_type
