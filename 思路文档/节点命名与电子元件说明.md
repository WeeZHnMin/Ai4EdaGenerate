# 节点命名与电子元件说明

## 1. 文档目的

这份文档用于解释当前数据集里常见的英文节点名、缩写和电子元件类型，方便在不具备电子工程背景的情况下理解这些数据到底在表达什么。

这里重点回答三个问题：

- 这个英文节点名是什么意思
- 它对应的是什么元件或电气节点
- 它在电路里通常起什么作用

## 2. 先理解两个概念

### 2.1 什么是“节点名”

例如：

- `R1`
- `C1`
- `VIN`
- `GND`
- `LED1`

这些名字是原理图里对某个元件、端点或特殊电气点的标记。

### 2.2 什么是“元件类型”

例如：

- `resistor`
- `capacitor`
- `diode`
- `power`
- `ground`

这些是程序里用的类型标签，用来告诉模型“这个节点属于哪一类”。

也就是说：

- `R1` 是节点名
- `resistor` 是它的类型

## 3. 最常见节点名对照

### 3.1 电源、地、输入、输出

| 节点名 | 大致含义 | 简单说明 |
|---|---|---|
| `VCC` | 电源 | 很常见的正电源名称，可以理解为“给电路供电的电” |
| `VDD` | 电源 | 也是正电源名称，和 `VCC` 类似，只是命名习惯不同 |
| `VIN` | 输入电压 / 输入端 | 表示电路输入，一般是信号或电源从这里进入 |
| `VOUT` | 输出电压 / 输出端 | 表示电路输出，一般是处理后的结果从这里出去 |
| `OUT` | 输出端 | 和 `VOUT` 类似，更通用 |
| `INPUT` | 输入端 | 和 `VIN` 类似，更偏通用命名 |
| `OUTPUT` | 输出端 | 和 `OUT` 类似 |
| `SIG_IN` | 信号输入 | 强调这是“信号”输入，不一定是供电脚 |
| `SIG_OUT` | 信号输出 | 强调这是“信号”输出 |
| `PWR_IN` | 电源输入 | 明确表示这里是供电入口 |
| `SUPPLY` | 电源 | 通用电源节点名 |
| `GND` | 地 | 电路中的参考地，可以理解为“零电位参考点” |

简单理解：

- `VCC`、`VDD`、`SUPPLY` 这类通常表示“电从哪里来”
- `VIN`、`INPUT`、`SIG_IN` 这类通常表示“信号从哪里进来”
- `OUT`、`VOUT`、`OUTPUT` 这类通常表示“结果从哪里出去”
- `GND` 表示“回路参考点”

## 4. 最常见元件命名规则

很多原理图里的名字不是随便起的，而是有前缀规则。

### 4.1 电阻

| 节点名 | 含义 |
|---|---|
| `R1`, `R2` | 第 1 个、第 2 个电阻 |
| `R_LIMIT` | 限流电阻 |
| `R_PULLUP` | 上拉电阻 |
| `R_PULLDOWN` | 下拉电阻 |
| `R_FILTER` | 滤波电阻 |
| `R_LOAD` | 负载电阻 |

说明：

电阻的英文是 `resistor`。

它的作用一般是：

- 限制电流
- 分压
- 上拉或下拉信号
- 参与滤波

最容易记住的点：

看到名字以 `R` 开头，基本就可以理解为“电阻”。

### 4.2 电容

| 节点名 | 含义 |
|---|---|
| `C1`, `C2` | 第 1 个、第 2 个电容 |
| `C_FILTER` | 滤波电容 |
| `C_DECOUPLE` | 去耦电容 |
| `C_BYPASS` | 旁路电容 |
| `C_SMOOTH` | 平滑电容 |

说明：

电容的英文是 `capacitor`。

它常见作用是：

- 滤波
- 去耦
- 储能
- 平滑电压波动

最容易记住的点：

看到名字以 `C` 开头，通常就是“电容”。

### 4.3 电感

| 节点名 | 含义 |
|---|---|
| `L1` | 第 1 个电感 |
| `L_FILTER` | 滤波电感 |
| `LMAIN` | 主电感 |

说明：

电感的英文是 `inductor`。

它常见作用是：

- 滤波
- 抑制高频噪声
- 与电容一起构成 LC 滤波网络

最容易记住的点：

看到名字以 `L` 开头，通常是“电感”。

### 4.4 二极管

| 节点名 | 含义 |
|---|---|
| `D1` | 第 1 个二极管 |
| `D_RECT` | 整流二极管 |
| `D_PROTECT` | 保护二极管 |
| `D_REV` | 反接保护二极管 |
| `D_HALF` | 半波整流用二极管 |

说明：

二极管的英文是 `diode`。

常见作用：

- 整流
- 防反接
- 单向导通
- 保护电路

最容易记住的点：

看到名字以 `D` 开头，通常是“二极管”。

### 4.5 发光二极管

| 节点名 | 含义 |
|---|---|
| `LED1` | 第 1 个 LED |
| `LED_STATUS` | 状态指示灯 |
| `LED_PWR` | 电源指示灯 |
| `LED_DRV` | 被驱动的 LED |

说明：

LED 也是二极管的一种，全称是发光二极管。

它的主要作用很简单：

- 亮灯指示状态

最容易记住的点：

看到 `LED`，就把它理解为“小灯”。

### 4.6 按键和开关

| 节点名 | 含义 |
|---|---|
| `KEY1` | 第 1 个按键 |
| `BTN1` | 第 1 个按钮 |
| `SW1` | 第 1 个开关 |

说明：

- `button` 表示按键
- `switch` 表示开关

它们的作用通常是：

- 控制信号拉高或拉低
- 产生开关状态输入

## 5. 当前数据里用到的类型说明

下面是程序里真正使用到的 `type` 字段解释。

### 5.1 `power`

表示电源节点。

对应常见名字：

- `VCC`
- `VDD`
- `SUPPLY`
- `PWR_IN`

简单理解：

这是“供电的地方”。

### 5.2 `ground`

表示地节点。

对应常见名字：

- `GND`

简单理解：

这是“参考地”，很多回路最终会接到这里。

### 5.3 `input`

表示输入节点。

对应常见名字：

- `VIN`
- `INPUT`
- `SIG_IN`
- `AC_IN`

简单理解：

这是“外部给电路的输入”。

### 5.4 `output`

表示输出节点。

对应常见名字：

- `OUT`
- `VOUT`
- `OUTPUT`
- `DC_OUT`

简单理解：

这是“电路处理完以后输出的结果”。

### 5.5 `junction`

表示中间连接节点。

对应常见名字：

- `MID`
- `NODE1`
- `KEY_SIG`
- `BTN_NODE`

简单理解：

这不是一个具体元件，更像是“中间连接点”。

### 5.6 `resistor`

表示电阻。

对应常见名字：

- `R1`
- `R2`
- `R_PULLUP`
- `R_FILTER`

### 5.7 `capacitor`

表示电容。

对应常见名字：

- `C1`
- `C_FILTER`
- `C_DECOUPLE`

### 5.8 `inductor`

表示电感。

对应常见名字：

- `L1`
- `L_FILTER`

### 5.9 `diode`

表示普通二极管。

对应常见名字：

- `D1`
- `D_RECT`
- `D_PROTECT`

### 5.10 `led`

表示发光二极管。

对应常见名字：

- `LED1`
- `LED_STATUS`

### 5.11 `button`

表示按键。

对应常见名字：

- `KEY1`
- `BTN1`

### 5.12 `switch`

表示开关。

对应常见名字：

- `SW1`

### 5.13 `load`

表示负载。

对应常见名字：

- `LOAD`
- `RL`
- `RLOAD`
- `DEVICE`

简单理解：

“负载”就是这个电路最终要带动、供电或者驱动的对象。

## 6. 常见缩写的直观解释

### 6.1 `VIN`

可以理解成“输入端”。

### 6.2 `VOUT`

可以理解成“输出端”。

### 6.3 `GND`

可以理解成“地”。

### 6.4 `MID`

可以理解成“中间点”。

### 6.5 `PULLUP`

上拉。

意思是：通过一个电阻把某个信号默认拉到高电平或电源。

### 6.6 `PULLDOWN`

下拉。

意思是：通过一个电阻把某个信号默认拉到低电平或地。

### 6.7 `FILTER`

滤波。

### 6.8 `DECOUPLE`

去耦。

可以理解成“给电源做局部稳定和抗干扰”。

### 6.9 `BYPASS`

旁路。

通常和电容有关，用来让高频噪声更容易流到地。

### 6.10 `RECT`

整流。

一般表示把交流变成更接近直流的过程。

## 7. 结合数据看一个例子

例如这样一条数据里的节点：

```json
{
  "components": {
    "VIN": {"type": "input", "x": 10, "y": 20},
    "R1": {"type": "resistor", "x": 40, "y": 20},
    "OUT": {"type": "output", "x": 70, "y": 20},
    "C1": {"type": "capacitor", "x": 70, "y": 50},
    "GND": {"type": "ground", "x": 70, "y": 80}
  }
}
```

可以这样理解：

- `VIN`：输入从这里进来
- `R1`：一个电阻
- `OUT`：处理后的输出点
- `C1`：一个电容
- `GND`：地

如果连接关系是：

```json
{
  "edge1": ["VIN", "R1"],
  "edge2": ["R1", "OUT"],
  "edge3": ["OUT", "C1"],
  "edge4": ["C1", "GND"]
}
```

那么这大致表示：

- 输入先进电阻
- 电阻后面是输出点
- 输出点再通过一个电容接地

这就是一个很典型的 RC 低通滤波结构。

## 8. 最简单的记忆方法

你不用一开始把所有元件都学会，只要先记住这些：

- `R` 开头：电阻
- `C` 开头：电容
- `L` 开头：电感
- `D` 开头：二极管
- `LED` 开头：发光二极管
- `VCC` / `VDD`：电源
- `GND`：地
- `VIN`：输入
- `OUT` / `VOUT`：输出
- `KEY` / `BTN`：按键
- `SW`：开关

先能看懂这些，已经足够理解当前这批数据里的大部分内容。

## 9. 一句话总结

当前数据集里的英文节点名并不是随意命名，而是电子原理图里很常见的一套缩写规则。

你可以先把它们理解成：

- 电从哪里来
- 经过什么元件
- 从哪里输出
- 有没有接地

只要能读懂这一层，后面再看模板、坐标和连线，就不会太陌生了。
